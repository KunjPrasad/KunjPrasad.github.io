Personal notes

Whenever reading any book, I've also felt the ability to add my own thoughts as missing. In this book, a framework has been added enabling readers to add their own views, comments, followup, etc. I would highly encourage readers to make their comment file available to other via their own Github repo. All paragraphs in the html book have been given a unique id. When the corresponding "Add note" button is clicked, an empty note is added. Modifying the text to a non empty value updates the text when focus is removed from the note box. If the note text is modified to empty, then the note is deleted.

    WORDS OF CAUTION:
  • The notes are not saved on the webpage hosting the html book. The readers must remember to save the notes before navigating away from the page, else the notes will be lost. The saved notes can be uploaded back when reader visits the page at a later time. The saved note file name is formatted to help identify that the notes belong to this book, and also contains a timestamp. For those interested, I want to mention that the saved note-file stored data in json format.
  • If applying someone else's notes, realize that you will be vulnerable to XSS attack. Use caution and your judgement.
