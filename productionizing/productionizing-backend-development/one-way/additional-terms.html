Additional terms

This page lists the various terms that relate to request handling but will not be used in future sections. Like the glossary items, it is good to know about them and what they do. However, it is very unlikely that they will be modified when developing custom backend code, and so, they don't cause any design concerns via a "n-way interaction" that must be kept in mind when coding.

Syncronous vs asynchronous request handling

This is not async made in JS, or async inside the code.. but async when initiating request processing by server. ASGI decomposes protocols into a series of events that an application must receive and react to, and events the application might send in response. For HTTP, this is as simple as receiving two events in order - http.request and http.disconnect, and sending the corresponding event messages back. For something like a WebSocket, it could be more like receiving websocket.connect, sending a websocket.send, receiving a websocket.receive, and finally receiving a websocket.disconnect. Because these messages could be sent over a network, they need to be serializable.
